## 1️⃣ **예외란 무엇인가?**

프로그램을 만들다 보면 예상치 못한 상황들이 발생합니다. 사용자가 숫자를 입력해야 하는데 문자를 입력한다거나, 0으로 나누기를 시도한다거나, 존재하지 않는 파일을 열려고 할 때 등입니다.

이런 상황에서 파이썬은 **예외(Exception)** 를 발생시키고, 아무 조치도 취하지 않으면 프로그램이 그 자리에서 멈춰버립니다. 사용자 입장에서는 매우 당황스러운 경험이죠.

### **예외가 발생하는 대표적인 상황들**

**1. 타입 오류 (ValueError)**

```python
# 숫자를 입력받으려고 했는데...
age = int(input("나이: "))
# 사용자가 "스물다섯"이라고 입력하면?
# ValueError: invalid literal for int() with base 10: '스물다섯'
# 프로그램 중단! ❌
```

**2. 0으로 나누기 (ZeroDivisionError)**

```python
# 평균을 계산하려고 했는데...
total = 100
count = 0
average = total / count
# ZeroDivisionError: division by zero
# 프로그램 중단! ❌
```

**3. 파일이 없음 (FileNotFoundError)**

```python
# 파일을 열려고 했는데...
file = open("없는파일.txt", "r")
# FileNotFoundError: No such file or directory: '없는파일.txt'
# 프로그램 중단! ❌
```

**4. 리스트 인덱스 오류 (IndexError)**

```python
# 리스트의 항목에 접근하려고 했는데...
fruits = ["사과", "바나나", "오렌지"]
print(fruits[10])
# IndexError: list index out of range
# 프로그램 중단! ❌
```

### **예외 처리의 필요성**

상상해보세요. 여러분이 만든 쇼핑몰 프로그램에서 사용자가 실수로 잘못된 값을 입력했는데 프로그램이 갑자기 멈춰버린다면? 사용자는 다시는 그 프로그램을 사용하지 않을 것입니다.

```
예외 처리가 없을 때 vs 있을 때

❌ 예외 처리 없음              ✅ 예외 처리 있음
┌─────────────────┐          ┌─────────────────┐
│ 나이: 스물다섯   │          │ 나이: 스물다섯   │
│ [오류 발생]     │          │ ⚠️ 숫자로 입력  │
│ 프로그램 종료   │          │ 나이: 25        │
│                 │          │ ✓ 정상 진행     │
└─────────────────┘          └─────────────────┘

사용자: 😡                   사용자: 😊
"이게 뭐야!"                 "친절하네!"
```

예외 처리를 하면:

- 프로그램이 중단되지 않고 계속 실행
- 사용자에게 친절한 오류 메시지 제공
- 문제를 수정하고 다시 시도할 기회 제공
- 전문적이고 안정적인 프로그램 구현

## 2️⃣ **try-except 기본 - 예외 잡기**

예외가 발생할 수 있는 코드를 안전하게 감싸는 방법이 바로 `try-except` 문입니다. 마치 안전망을 치는 것과 같습니다.

### **try-except의 작동 원리**

```python
try:
    # 위험할 수 있는 코드를 여기에
    # "이 코드를 실행해봐. 문제가 생길 수 있어."
    위험한_작업()
  
except:
    # 예외가 발생했을 때 실행할 코드
    # "문제가 생기면 이렇게 처리해."
    안전한_대응()
```

```
try-except 흐름도

    시작
     │
     ▼
┌────────┐
│  try   │ 코드 실행 시도
└───┬────┘
    │
    ├─── 정상 실행 ──→ 계속 진행
    │
    └─── 예외 발생 ──→ except 블록 실행
                         │
                         ▼
                      계속 진행
```

### **간단한 예제로 이해하기**

**예외 처리가 없는 코드:**

```python
# 위험한 코드
number = int(input("숫자를 입력하세요: "))
print(f"입력한 숫자: {number}")
print("프로그램 계속 실행...")

# 사용자가 "abc"를 입력하면?
# ValueError 발생!
# 프로그램 즉시 종료!
# "프로그램 계속 실행..."은 출력되지 않음
```

**예외 처리가 있는 코드:**

```python
# 안전한 코드
try:
    number = int(input("숫자를 입력하세요: "))
    print(f"입력한 숫자: {number}")
except:
    print("❌ 오류! 숫자가 아닙니다.")
    print("💡 숫자로 다시 입력해주세요.")

print("프로그램 계속 실행...")

# 사용자가 "abc"를 입력하면?
# except 블록이 실행됨
# 친절한 메시지 출력
# 프로그램은 계속 실행! ✅
```

**실행 결과 비교:**

```
[정상 입력 - 25]
숫자를 입력하세요: 25
입력한 숫자: 25
프로그램 계속 실행...

[잘못된 입력 - abc]
숫자를 입력하세요: abc
❌ 오류! 숫자가 아닙니다.
💡 숫자로 다시 입력해주세요.
프로그램 계속 실행...
```

프로그램이 중단되지 않고 계속 실행됩니다!

### **여러 번 시도하기 - while과 결합**

사용자가 올바른 입력을 할 때까지 계속 물어보는 것이 더 친절합니다:

```python
# 올바른 입력을 받을 때까지 반복
while True:
    try:
        age = int(input("나이를 입력하세요: "))
        print(f"✓ 입력 성공! {age}세")
        break  # 성공하면 반복 종료
    
    except:
        print("❌ 숫자만 입력해주세요.")
        print("💡 예: 25\n")
        # break가 없으므로 다시 입력받음

print("다음 단계로 진행합니다...")
```

**실행 예시:**

```
나이를 입력하세요: 스물다섯
❌ 숫자만 입력해주세요.
💡 예: 25

나이를 입력하세요: twenty
❌ 숫자만 입력해주세요.
💡 예: 25

나이를 입력하세요: 25
✓ 입력 성공! 25세
다음 단계로 진행합니다...
```

### **try-except를 언제 사용하나요?**

예외 처리는 **"문제가 생길 수 있는 곳"** 에 사용합니다:

1. **사용자 입력**: 사용자는 예상 못한 값을 입력할 수 있음
2. **파일 작업**: 파일이 없거나, 권한이 없거나, 손상될 수 있음
3. **네트워크 통신**: 인터넷 연결이 끊기거나, 서버가 응답 안 할 수 있음
4. **외부 데이터**: JSON, CSV 등 데이터 형식이 잘못될 수 있음
5. **계산**: 0으로 나누기, 음수의 제곱근 등

⚠️ **주의**: 모든 코드를 try-except로 감싸면 안 됩니다! 진짜 버그를 숨길 수 있습니다. 문제가 생길 수 있는 부분만 선택적으로 사용하세요.

### **예제 1: 안전한 계산기**

0으로 나누기 오류를 처리하는 계산기입니다.

```python
# 안전한 나눗셈 계산기
print("➗" + "=" * 38 + "➗")
print("   나눗셈 계산기")
print("➗" + "=" * 38 + "➗")

while True:
    print("\n" + "=" * 40)
  
    try:
        num1 = float(input("첫 번째 숫자: "))
        num2 = float(input("두 번째 숫자: "))
    
        result = num1 / num2
    
        print("\n" + "-" * 40)
        print(f"{num1} ÷ {num2} = {result}")
        print("-" * 40)
  
    except ZeroDivisionError:
        print("\n❌ 0으로 나눌 수 없습니다!")
  
    except ValueError:
        print("\n❌ 숫자를 입력해주세요!")
  
    except:
        print("\n❌ 알 수 없는 오류가 발생했습니다!")
  
    # 계속할지 물어보기
    choice = input("\n계속하시겠습니까? (Y/N): ")
    if choice != "Y" and choice != "y":
        print("\n계산기를 종료합니다.")
        break
```

## 3️⃣ **다양한 예외 유형 - 정확하게 잡기**

지금까지는 `except:` 만 사용해서 모든 예외를 한꺼번에 잡았습니다. 하지만 예외 종류에 따라 다르게 처리하고 싶다면 어떻게 해야 할까요?

### **왜 예외 유형을 구분해야 할까요?**

```python
# 모든 예외를 똑같이 처리 (권장 안 함)
try:
    age = int(input("나이: "))
    result = 100 / age
except:
    print("오류가 발생했습니다!")  # 어떤 오류인지 모름!
```

사용자가 "스물"을 입력했을 때와 "0"을 입력했을 때, 오류 원인이 다릅니다. 하지만 같은 메시지만 보여주면 사용자는 무엇이 잘못됐는지 알 수 없습니다.

### **주요 예외 유형**

파이썬의 예외는 종류가 매우 다양합니다. 자주 만나는 것들을 알아봅시다:

```
📋 자주 만나는 예외들

🔢 숫자 관련
├─ ValueError        : 잘못된 값 (int("abc"))
├─ ZeroDivisionError : 0으로 나누기
└─ OverflowError     : 숫자가 너무 큼

📁 파일 관련
├─ FileNotFoundError : 파일이 없음
├─ PermissionError   : 권한 없음
└─ IsADirectoryError : 파일이 아닌 폴더

📚 자료구조 관련
├─ IndexError        : 인덱스 범위 초과
├─ KeyError          : 딕셔너리 키 없음
├─ KeyboardInterrupt : Ctrl+C로 중단
└─ TypeError         : 타입 불일치

🌐 기타
├─ AttributeError    : 속성/메서드 없음
├─ ImportError       : 모듈을 찾을 수 없음
└─ NameError         : 변수 이름이 정의 안 됨
```

### **특정 예외만 처리하기**

예외 유형을 지정하면 그 예외만 잡을 수 있습니다:

```python
# ValueError만 처리
try:
    age = int(input("나이: "))
    print(f"당신은 {age}세입니다.")
  
except ValueError:
    print("❌ 숫자를 입력해주세요!")
    print("💡 예: 25")
```

이제 `ValueError`(잘못된 값)만 잡고, 다른 오류는 프로그램을 중단시킵니다. 이것이 더 안전합니다. 예상하지 못한 오류를 숨기지 않기 때문입니다.

### **여러 예외를 다르게 처리하기**

각 예외마다 다른 메시지를 보여줄 수 있습니다:

```python
# 여러 예외를 구분해서 처리
try:
    # 사용자 입력
    numbers = [1, 2, 3, 4, 5]
    index = int(input("인덱스 (0-4): "))
  
    # 인덱스로 접근
    print(f"선택한 숫자: {numbers[index]}")
  
    # 계산
    result = 100 / index
    print(f"계산 결과: {result}")

except ValueError:
    # int() 변환 실패
    print("❌ 숫자를 입력해주세요!")
  
except IndexError:
    # 리스트 범위 벗어남
    print("❌ 0부터 4 사이의 숫자를 입력해주세요!")
    print(f"💡 사용 가능한 인덱스: 0, 1, 2, 3, 4")
  
except ZeroDivisionError:
    # 0으로 나누기
    print("❌ 0은 입력할 수 없습니다!")
    print("💡 1 이상의 숫자를 입력해주세요.")
```

**실행 예시:**

```
[경우 1: 문자 입력]
인덱스 (0-4): abc
❌ 숫자를 입력해주세요!

[경우 2: 범위 초과]
인덱스 (0-4): 10
❌ 0부터 4 사이의 숫자를 입력해주세요!
💡 사용 가능한 인덱스: 0, 1, 2, 3, 4

[경우 3: 0 입력]
인덱스 (0-4): 0
선택한 숫자: 1
❌ 0은 입력할 수 없습니다!
💡 1 이상의 숫자를 입력해주세요.

[경우 4: 정상]
인덱스 (0-4): 2
선택한 숫자: 3
계산 결과: 50.0
```

### **여러 예외를 같은 방식으로 처리하기**

때로는 여러 예외를 똑같이 처리하고 싶을 때가 있습니다. 괄호로 묶으면 됩니다:

```python
try:
    # 파일 열기 시도
    with open(filename, "r") as file:
        content = file.read()
    
except (FileNotFoundError, PermissionError):
    # 두 예외를 같은 방식으로 처리
    print("❌ 파일을 열 수 없습니다!")
    print("💡 파일 이름과 권한을 확인해주세요.")
```




### **예제 2: 파일 읽기**

파일 관련 예외를 처리하는 프로그램입니다.

```python
# 안전한 파일 읽기 프로그램
print("📄" + "=" * 38 + "📄")
print("   파일 읽기")
print("📄" + "=" * 38 + "📄")

while True:
    filename = input("\n파일 이름: ")
  
    if filename == "q" or filename == "Q":
        print("\n프로그램을 종료합니다.")
        break
  
    try:
        with open(filename, "r", encoding="utf-8") as file:
            content = file.read()
    
        print("\n" + "=" * 40)
        print(f"📖 {filename}")
        print("=" * 40)
        print(content)
        print("=" * 40)
  
    except FileNotFoundError:
        print(f"\n❌ '{filename}' 파일을 찾을 수 없습니다!")
        print("💡 파일 이름을 확인해주세요.")
  
    except PermissionError:
        print(f"\n❌ '{filename}' 파일을 읽을 권한이 없습니다!")
  
    except UnicodeDecodeError:
        print(f"\n❌ '{filename}' 파일의 인코딩이 올바르지 않습니다!")
        print("💡 텍스트 파일이 아닐 수 있습니다.")
  
    except:
        print("\n❌ 파일을 읽는 중 오류가 발생했습니다!")
```

## 4️⃣ **예외 메시지 받기 - 더 자세한 정보**

예외가 발생하면 파이썬은 "무엇이 잘못되었는지" 설명하는 메시지를 함께 보내줍니다. 이 메시지를 받아서 사용자에게 보여주거나, 로그에 기록할 수 있습니다.

### **as 키워드로 예외 정보 받기**

```python
try:
    number = int(input("숫자: "))
except ValueError as e:
    # e 변수에 예외 정보가 담김
    print(f"오류 발생!")
    print(f"상세 내용: {e}")
```

**실행 예시:**

```
숫자: 안녕
오류 발생!
상세 내용: invalid literal for int() with base 10: '안녕'
```


### **실전 예제: 딕셔너리 검색**

```python
# 과일 가격 검색 프로그램 (개선된 오류 처리)
fruits = {
    "사과": 3000,
    "바나나": 2000,
    "오렌지": 4000
}

print("🍎 과일 가격표")
print("등록된 과일:", ", ".join(fruits.keys()))

fruit = input("\n검색할 과일: ")

try:
    price = fruits[fruit]
    print(f"✓ {fruit}: {price:,}원")
  
except KeyError as e:
    print(f"❌ 등록되지 않은 과일입니다!")
    print(f"💡 찾으려던 과일: {e}")  # 'KeyError의 메시지'
  
    # 비슷한 이름 추천
    similar = [f for f in fruits.keys() if fruit[0] in f]
    if similar:
        print(f"💡 혹시 이것을 찾으시나요? {', '.join(similar)}")
```

## 5️⃣ **finally - 항상 실행되는 코드**

`finally` 블록은 예외 발생 여부와 **관계없이 항상 실행**됩니다. 마치 "어떤 일이 있어도 반드시 해야 하는 일"을 지정하는 것입니다.

### **finally가 필요한 이유**

파일을 열었으면 닫아야 하고, 데이터베이스에 연결했으면 연결을 끊어야 하고, 자원을 빌렸으면 반납해야 합니다. 예외가 발생하든 안 하든 이런 "정리 작업"은 반드시 해야 합니다.

```
try-except-finally 실행 흐름

              시작
               │
           ┌───▼───┐
           │  try  │ 코드 실행
           └───┬───┘
               │
        ┌──────┴──────┐
     정상실행      예외발생
        │              │
        │          ┌───▼────┐
        │          │ except │ 예외 처리
        │          └───┬────┘
        │              │
        └──────┬───────┘
               │
           ┌───▼────┐
           │finally │ 항상 실행!
           └───┬────┘
               │
              종료
```

### **기본 예제**

```python
try:
    print("1. 작업 시작")
    result = 10 / 0  # 예외 발생!
    print("2. 작업 완료")  # 실행 안 됨
  
except ZeroDivisionError:
    print("3. 오류 처리")
  
finally:
    print("4. 정리 작업 (항상 실행)")
  
print("5. 프로그램 계속")
```

**실행 결과:**

```
1. 작업 시작
3. 오류 처리
4. 정리 작업 (항상 실행)
5. 프로그램 계속
```

"2. 작업 완료"는 실행되지 않았지만, "4. 정리 작업"은 실행되었습니다!

### **실전 예제: 파일 작업**

```python
# 파일 작업 with finally
file = None  # 변수 미리 선언

try:
    print("📂 파일 열기 시도...")
    file = open("data.txt", "r")
    content = file.read()
    print(f"✓ 파일 읽기 성공!")
    print(content)
  
except FileNotFoundError:
    print("❌ 파일을 찾을 수 없습니다!")
  
finally:
    # 파일이 열렸든 안 열렸든, 열려있다면 닫기
    if file:
        file.close()
        print("🔒 파일 닫기 완료")
    print("정리 작업 완료")
```

💡 **참고**: `with` 문을 사용하면 자동으로 파일을 닫아주므로 더 간편합니다. 하지만 `finally`의 개념을 이해하는 것이 중요합니다.

## 6️⃣ **예외 직접 발생시키기 (raise)**

지금까지는 파이썬이 자동으로 발생시키는 예외를 처리하는 방법을 배웠습니다. 하지만 때로는 우리가 직접 예외를 발생시켜야 할 때가 있습니다.

### **왜 일부러 예외를 발생시킬까요?**

프로그램의 규칙을 강제하기 위해서입니다. 예를 들어:

- 나이는 0보다 커야 한다
- 비밀번호는 6자 이상이어야 한다
- 파일 크기는 10MB를 넘으면 안 된다

이런 규칙을 어겼을 때 예외를 발생시켜서 프로그램을 안전하게 만들 수 있습니다.

### **raise 키워드 사용법**

```python
# 기본 형태
raise 예외종류("오류 메시지")

# 예시
raise ValueError("나이는 0보다 커야 합니다")
raise TypeError("문자열이 아닙니다")
raise FileNotFoundError("파일을 찾을 수 없습니다")
```

### **간단한 예제**

```python
def divide(a, b):
    """두 수를 나누는 함수"""
  
    # 0으로 나누기를 사전에 차단
    if b == 0:
        raise ValueError("0으로 나눌 수 없습니다!")
  
    return a / b

# 사용
try:
    result = divide(10, 0)
    print(result)
  
except ValueError as e:
    print(f"❌ {e}")
```

**실행 결과:**

```
❌ 0으로 나눌 수 없습니다!
```

### **입력 검증에 활용하기**

```python
def create_user(username, age):
    """사용자 생성 함수"""
  
    # 아이디 검증
    if not username:
        raise ValueError("아이디를 입력해주세요")
  
    if len(username) < 3:
        raise ValueError("아이디는 3자 이상이어야 합니다")
  
    # 나이 검증
    if not isinstance(age, int):
        raise TypeError("나이는 숫자여야 합니다")
  
    if age < 0:
        raise ValueError("나이는 0 이상이어야 합니다")
  
    if age > 150:
        raise ValueError("나이가 너무 많습니다")
  
    # 모든 검증 통과
    return {
        "username": username,
        "age": age
    }

# 사용 예시
try:
    user = create_user("ab", 25)
    print(f"✓ 사용자 생성: {user}")
  
except ValueError as e:
    print(f"❌ {e}")
  
except TypeError as e:
    print(f"❌ {e}")
```

**실행 결과:**

```
❌ 아이디는 3자 이상이어야 합니다
```

## 📝 **핵심 개념 정리**

### **예외(Exception)**

프로그램 실행 중 발생하는 오류입니다. 처리하지 않으면 프로그램이 중단됩니다.

```python
# 주요 예외 종류
ValueError          # 잘못된 값
ZeroDivisionError   # 0으로 나누기
FileNotFoundError   # 파일 없음
IndexError          # 인덱스 범위 초과
KeyError            # 딕셔너리 키 없음
```

### **try-except 기본**

```python
try:
    # 예외가 발생할 수 있는 코드
    risky_code()
except ValueError:
    # ValueError 처리
    handle_error()
except KeyError as e:
    # 예외 정보 받기
    print(f"오류: {e}")
```

### **완전한 구조**

```python
try:
    code()              # 실행 시도
except ValueError:      # 예외 발생시
    handle_error()
else:                   # 예외 없을 때만
    success()
finally:                # 항상 실행
    cleanup()
```

### **raise - 예외 발생시키기**

```python
def set_age(age):
    if age < 0:
        raise ValueError("나이는 0 이상이어야 합니다")
    return age
```

### **기억할 핵심**

- try-except로 예외를 안전하게 처리
- 구체적인 예외 타입 지정 (ValueError, KeyError 등)
- as로 예외 메시지 받기
- finally는 항상 실행 (정리 작업용)
- raise로 직접 예외 발생 가능

### **과제 1: 안전한 리스트 접근**

인덱스 입력을 받아 리스트 항목을 출력하되, 잘못된 입력을 처리하세요.

```python
# 힌트
fruits = ["사과", "바나나", "오렌지", "포도", "딸기"]

try:
    index = int(input("인덱스: "))
    print(fruits[index])
except ValueError:
    print("숫자를 입력하세요")
except IndexError:
    print("인덱스 범위를 벗어났습니다")
```

### **과제 2: 파일 백업**

파일을 읽어서 백업 파일로 복사하되, 모든 예외를 처리하세요.

```python
# 힌트
try:
    source = input("원본 파일: ")
    with open(source, "r") as f:
        content = f.read()
  
    backup = source + ".backup"
    with open(backup, "w") as f:
        f.write(content)
  
    print(f"백업 완료: {backup}")
  
except FileNotFoundError:
    print("파일을 찾을 수 없습니다")
except PermissionError:
    print("권한이 없습니다")
finally:
    print("작업 종료")
```
